# Отчёт по ускорению модели YOLOv11n с помощью квантизации INT8

## 1. Выбор базовой модели

**Базовая модель:** YOLOv11n.  
**Задача:** Обнаружение объектов на изображениях. Модель обучена на наборе данных COCO и способна детектировать 80 классов объектов.  
**Сценарий использования:** Инференс на серверных GPU или встраиваемых устройствах (NVIDIA Jetson). Требуется высокая скорость обработки при сохранении приемлемого качества детекции.  
**Аппаратная платформа:** NVIDIA RTX 4070 ti.  

**Ключевые bottlenecks, выявленные ранее:**
- Высокая вычислительная нагрузка из-за большого количества свёрточных операций (FP32).
- Значительное потребление памяти видеокарты, что ограничивает размер батча или возможность запуска на устройствах с ограниченной памятью.
- latency выше требуемой для real-time приложений.

## 2. Выбор и реализация метода ускорения

**Выбранный метод:** Квантизация  модели из FP32 в INT8 с помощью TensorRT c калибровкой.  
**Почему метод подходит:**
- Модель YOLOv11n содержит преимущественно свёрточные слои, которые хорошо поддаются квантизации без критической потери точности.
- TensorRT  предоставляет эффективную реализацию INT8 inference на GPU, используя аппаратные возможности (тензорные ядра) для ускорения.
- TensorRT  движок NVIDIA подходит под любые чипы под их производством, можно выполнить калибровку и использовать калиб файл для конвертации на любом чипе.

**Ограничения:**
- Квантизация может привести к падению точности (mAP), особенно если распределение активаций сложное. Для минимизации потерь применяется калибровка на репрезентативном наборе данных (COCO val).

## 3. Алгоритм применения метода

Для реализации квантизации используется скрипт на Python. Скрипт выполняет следующие шаги:

1. **Загрузка ONNX модели** 
2. **Создание даталоадера** для калибровки: итерация по изображениям из COCO val, предобработка (letterbox, нормализация) в соответствии с требованиями модели.
3. **Конфигурация TensorRT builder**:
4. **Запуск калибровки**: калибратор обрабатывает батчи изображений, собирает статистики распределений активаций и вычисляет масштабирующие коэффициенты (scales) для каждого слоя.
5. **Построение INT8 engine**: TensorRT компилирует сеть с учётом собранных калибровочных данных.
6. **Сохранение engine и калиб файла** для последующего использования.

Калибровочный файл - файл с инструкциями по слоям для калибровки, его можно использовать в сочетании с `trtexec` на целевой платформе для того чтобы добиться такой же конфигурации графа модели, как после калибровки на данных. Обычно так и делают квантизацию: калибровку проводят на доступной гпу, получают `calib.txt` и затем конвертируют модель на целевой платформе.

**Настройки:**
- Количество калибровочных изображений: val coco 2017 5000 изображений достаточно для репрезентативной статистики.
- Алгоритм калибровки: EntropyCalibration2.
- CUDNN - оптимизатор

## 4. Проведение эксперимента

**Цель эксперимента:** Сравнить производительность и качество базовой FP32 модели и квантизованной INT8 модели на одинаковых условиях.

**Конфигурация среды:**
  - Ubuntu 22.04
  - CUDA 12.8
  - TensorRT 10.9.0.34
  - Python 3.10.12
- Данные: COCO 2017 val 5000 изображений

**Параметры запуска:**
- Измеряемые метрики:
  - mAP@0.5:0.95.
  - Задержка (latency) в миллисекундах на одно изображение.

Конфигурация конечной модели доступна в `cal.bin`.
## 5. Выбор и обоснование метрик

Для оценки качества модели используется **mAP@0.5:0.95** (mean Average Precision) — стандартная метрика для COCO, учитывающая точность при различных порогах Intersection over Union. Она релевантна для задачи обнаружения объектов и позволяет сравнить модели на равных.
В файле `val_metrics` приведены результаты валидации по всем метрикам, для упрощения в таблице я привожу только **mAP@0.5:0.95**

## 6. Сравнение модели до и после ускорения

В таблице представлены ожидаемые результаты (студент должен подставить свои реальные значения после эксперимента).

| Модель      | mAP@0.5:0.95 | Latency (ms) |
|----------|-------------|---------|
| FP32 (pt) | 39.4 | 5.6|
| INT8 (trt) | 35.6 (-3.8) | 1.8 (3.11x) |

**Интерпретация:**
- Квантизация в INT8 позволила ускорить инференс в **3.11 раза** при снижении mAP на **3.8** это примерно **-10 %** от исходного значения . На практике это сильно заметно, но надо учитывать, что замер происходил на COCO датасете, который сам по себе шумный, для калибровки использовалось не так много изображений, на практике количество классов не такое большое, вариация детекций сильно меньше и можно достичь меньшего ухудшения до **5%**.
## 7. Вывод о целесообразности подхода

Квантизация INT8 через TensorRT является **высокоэффективным методом ускорения** для YOLOv11n на GPU. Достигнуто значительное снижение задержки и потребления ресурсов при минимальной потере точности.

**Целесообразность применения:**
- Для production-сценариев, где критична скорость обработки (видеостриминг, реальное время) и допустимо небольшое падение качества, метод оправдан полностью.
- На платформах с ограниченной памятью (Jetson Nano, Xavier) INT8 модель позволяет запускать инференс с большим батчем или вообще становится единственно возможной.
- Если требуется максимально возможная точность следует оставаться на FP32/FP16.

**Ограничения:**  
Метод требует наличия калибровочного набора данных и долгой калибровки, размен качество-скорость 5-10%.


**Как запустить:**
```bash
git clone 
cd hw2/
docker build -t yolo11-quant .
docker run --gpus all -it --rm -v $(pwd):/workspace yolo11-quant
```
Внутри контейнера:
```bash
python onnx2trt8.py -m yolo11n.onnx -d datasets/coco/images/val2017/  -s 3,640,640 -o calib.bin -t int8
```

Измерить метрики
```bash
yolo detect val model=calib.engine data=coco.yaml split=val batch=1 
yolo detect val model=yolo11n.pt data=coco.yaml split=val batch=1 
```
